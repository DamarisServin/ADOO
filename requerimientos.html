Requerimientos

Requerimientos Funcionales

  • El sistema deberá guardar la información de cada banda o artista, es decir: nombre, género, ciudad de nacimiento, biografía y su sitio web. Si alguno de estos datos no es conocido, el dato se sustituirá por las siglas NA (No Disponible, por sus siglas en ingles).
  • El sistema organizará la información de los álbumes del artista o banda en el siguiente orden: nombre del álbum, año de lanzamiento, numero de pistas y estudio de grabación.
  • Se almacenará nombre, dirección, sitio web oficial y número telefónico de la compañía de grabación que produjo cada álbum del sistema. Si alguno de estos datos no es conocido, el dato se sustituirá por las siglas NA (No Disponible, por sus siglas en ingles).
  • Debe de mantener la información de las canciones del artista o banda, es decir, el nombre de la canción, su duración y el número de pista que tiene en el álbum correspondiente.
  • De cada álbum se quiere almacenar y mostrar su imagen de portada y contraportada.
  • El sistema mostrará si una banda o artista está influenciada por otras bandas o artistas. De estos se mostrará su nombre, ciudad de nacimiento y, en caso de ser un artista, su instrumento.
  • En la pantalla de “Acerca del Artista”, el sistema desplegará las bandas a las que el artista ha pertenecido. Si el artista no ha pertenecido a ninguna banda este apartado se omitirá.
  • El sistema debe mostrar si un músico es un artista invitado en alguna canción o álbum.
  • Si es el caso, se quiere tener una tablatura por canción que contenga fecha, URL (Localizador de Recursos Uniforme, por sus siglas en inglés), el traductor y el correo del traductor.
  • Las búsquedas en el sistema le permitirán al usuario encontrar canciones y artistas o bandas basadas en el nombre de las mismas. Si no se encuentra alguna canción o artista se desplegará el mensaje “No se encontró ningún resultado” .

Requerimientos No Funcionales

  • El sistema permitirá el acceso a un máximo de 50 usuarios de forma concurrente.
  • El registro de los cambios hechos en la plataforma se realizará de manera inmediata en la base de datos.
  • Los permisos de acceso al sistema podrán ser cambiados solamente por el administrador del sistema.
  • La configuración de los parámetros de instalación no debe requerir modificaciones al código fuente de la instalación.
  • El nuevo sistema o componente de software puede ser modificado para corregir fallos, mejorar su funcionamiento u otros atributos o adaptarse a cambios en el entorno.
  • Los componentes base o herramientas utilizadas para el diseño, construcción, implementación y la realización de pruebas, deberán tener más de 3 años en el mercado, soporte por parte del fabricante, tener fabricante reconocido.

Restricciones

  • El sistema web debe de ser compatible con todos los navegadores web Google Chrome, Opera, Microsoft Edge, Safari y Mozillla Firefox.
  • El sistema web deberá ser adaptable a cualquier tipo de dispositivo en el cual se pueda acceder a los navegadores antes mencionados (smartphones, computadoras y tabletas).
  • El sistema web debe mostrar los resultados usando la codificación de caracteres 8-bit Unicode Transformation Format (UTF-8).
  • El sistema web deberá tener un tiempo de carga mínimo de 2 segundos.
  • El sistema web debe almacenar un máximo de 300 artistas/bandas.
